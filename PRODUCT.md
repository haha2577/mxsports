# 球类赛事平台 — 产品规划文档

> 版本：v0.1  
> 创建时间：2026-03-01  
> 状态：规划阶段

---

## 一、产品定位

**一句话定位：** 专为羽毛球和网球玩家设计的、越用越懂你的运动成长伙伴。

**核心差异点：**
- 不只是赛事工具，而是一个持续积累球员数据的智能档案系统
- 通过比赛记录 + 视频 AI 分析，对每个球员的了解随时间不断加深
- 以工具为入口，最终建立垂直运动社交和商业生态

---

## 二、领域范围

| 领域 | 视觉风格 | 特点 |
|------|---------|------|
| 🏸 **羽毛球** | 清新绿色，轻盈简洁 | 室内竞技，节奏快 |
| 🎾 **网球** | 红土橙色，精英专业感 | 户外场地，技术性强 |

两个领域共用底层数据和功能架构，UI 风格有明显差异。

---

## 三、目标用户

| 用户类型 | 描述 | 核心诉求 |
|---------|------|---------|
| **普通球员** | 羽球 / 网球爱好者，有一定参赛频率 | 记录成长、找对手、提升技术 |
| **赛事组织者** | 球队队长、球馆、社区赛事发起人 | 高效组织比赛、管理球员 |
| **教练**（后期） | 持证或有经验的专业教练 | 获客、展示资历、管理学员 |

### 阶段一：工具阶段（当前重点）

**目标：** 把工具做到极致好用，靠工具留住用户、积累数据。

#### 1. 球员档案
- 创建个人资料（姓名、惯用手、水平分级）
- 手动录入历史比赛数据（解决冷启动问题）
- 球员积分 / 胜率 / 场次统计
- 比赛视频上传与关联

#### 2. 赛事报名
- 浏览公开赛事
- 报名 / 取消报名
- 报名状态追踪

#### 3. 组建球队
- 创建球队（队名、领域、成员邀请）
- 队伍成员管理
- 团体赛报名

#### 4. 组织比赛（赛事管理工具）
- 创建赛事（单打 / 双打 / 团体）
- 根据报名表自动生成赛制：
  - 循环赛（Round Robin）
  - 单淘汰 / 双淘汰（Knockout）
  - 分组循环 + 淘汰（Group Stage）
  - 羽毛球 / 网球专属赛制（如汤尤杯制、戴维斯杯制）
- 对阵签表自动生成
- 比分录入与实时更新
- 赛事结果存档

#### 5. AI 功能：智能视频剪辑（一期 AI 核心功能）

> 打球的人最大的痛点之一：录完视频后，精彩片段散落在几十分钟素材里，手动剪辑耗时，大多数人放弃。

**功能流程：**

1. **上传比赛视频** — 支持上传完整比赛录像，后台异步处理，完成后推送通知
2. **自动识别精彩片段** — 通过画面动作幅度、运动轨迹、音频特征（击球声、加油声）自动标记高潮时刻，用户可预览和微调
3. **人物识别 & 选择** — 自动识别视频中的人物，用户选择「要谁的片段」，系统只保留包含所选人物的精彩内容
4. **一键导出** — 合成个人精彩集锦，支持下载或直接分享

**技术路线（初期）：**

| 能力 | 方案 |
|------|------|
| 精彩片段检测 | 动作检测 + 音频峰值分析（FFmpeg + 视觉模型） |
| 人物识别 | 人脸检测 + 人物 ReID（跨镜头同一人追踪） |
| 首次确认机制 | 第一次用户手动框选确认是谁，后续自动记住 |
| 视频合成 | FFmpeg 剪辑拼接 |
| 存储 | 对象存储（腾讯云 COS / 阿里云 OSS） |

**连锁价值：** 用户上传视频 → 系统识别人物 → 自动关联球员档案 → AI 积累每个人的技术特征 → 为后续动作分析、技术指导打基础。看起来是剪辑工具，实际上是**数据飞轮的起点**。

**后续延伸（不急）：**
- 片段质量自动评分
- 自动加比分、球员名字水印
- 一键生成带背景音乐的精彩集锦模板

**阶段一关键指标：** 月活球员数、比赛场次、视频上传量、用户留存率

### 阶段二：社交阶段

**目标：** 基于工具积累的数据，建立人与人之间的连接。

#### 1. 约球 / 找搭档
- 基于水平、地区、时间智能匹配对手或搭档
- 发起约球邀请、接受匹配

#### 2. AI 专业指导
- 基于比赛历史给出个性化训练建议
- 视频 AI 深度分析（挥拍姿势、步伐、战术模式）
- 个人技术报告（强弱项、进步曲线）

#### 3. 圈子 / 社群
- 系统基于水平 / 地区 / 风格主动推荐合适的圈子
- 圈子内活动发布、讨论
- 同赛事参与者自动形成赛事社群

#### 4. 教练撮合（社交阶段后期）
- 教练入驻与资质认证
- 教练主页（履历、擅长方向、收费）
- 球员 ↔ 教练智能匹配推荐
- 预约 / 排课闭环

**阶段二关键指标：** 社交互动率、约球成功率、教练匹配量

### 阶段三：商业化阶段

**目标：** 基于深度用户画像，构建精准变现路径。

**商业模式：**

| 类型 | 具体形式 |
|------|---------|
| **增值服务** | 高级 AI 视频分析报告、深度技术诊断、专属训练计划 |
| **装备电商** | 基于球员画像精准推荐球拍 / 球鞋 / 护具，导购或自营 |
| **教练撮合抽佣** | 教练接单成功后平台收取服务费 |
| **赛事 SaaS** | 为大型赛事 / 球馆提供赛制管理工具（B 端付费） |
| **精准资讯** | 赛事资讯、技术文章定向推送（广告或订阅） |

**阶段三关键指标：** GMV、付费转化率、ARPU

## 五、重要设计原则

1. **工具先行** — 第一阶段不强调 AI，工具本身要足够好用，靠体验留人
2. **两类用户分开设计** — 「组织者」和「参与者」诉求不同，功能和界面分别对待
3. **付费功能早设计、晚收费** — 现在规划好高级功能，打「即将上线」标签，建立用户付费预期
4. **视频是核心数据入口** — 所有 AI 能力的基础，上传体验要做到极简
5. **分享即传播** — 精彩集锦天然适合分享，设计上要降低分享门槛

## 六、技术架构（现阶段）

- **前端：** 微信小程序（原生）→ 后续扩展 H5 / App
- **后端：** Django 6 + DRF，SQLite → 后续 PostgreSQL
- **视频处理：** FFmpeg + 第三方视觉 API（异步队列处理）
- **存储：** 腾讯云 COS 或阿里云 OSS
- **部署：** AWS EC2，Docker 容器化
